# Patient Portal Website Workspace

## Workspace Layout

- `website/`: active implementation workspace for SCRUM-9
- `mobile/`: placeholder for mobile app implementation
- `backend/`: placeholder for backend implementation
- `tests/`: shared top-level tests and validation reports

## Local Setup

1. `cd website`
2. `npm install`
3. `npm run dev`

## Validation Commands

- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`
- `npm run validate`

## Startup Verification

- Start the app with `npm run dev`
- Open `http://localhost:3000`
- Confirm baseline navigation and startup state rendering
