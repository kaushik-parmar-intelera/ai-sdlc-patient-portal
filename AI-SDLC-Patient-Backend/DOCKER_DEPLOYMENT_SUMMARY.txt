## ✅ Quick Start (Choose Your Path)

### Windows Users
```powershell
# Make scripts executable (PowerShell 7+)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# Start development
./docker-manage.ps1 dev-start

# View logs
./docker-manage.ps1 dev-logs api

# Stop when done
./docker-manage.ps1 dev-stop
```

### macOS/Linux Users
```bash
# Make script executable
chmod +x docker-manage.sh

# Start development
./docker-manage.sh dev-start

# View logs
./docker-manage.sh dev-logs api

# Stop when done
./docker-manage.sh dev-stop
```

### Everyone - Manual Commands
```bash
# Start development environment
docker-compose -f docker-compose.dev.yml up -d

# View logs (wait 30-40 seconds for database)
docker-compose -f docker-compose.dev.yml logs -f api

# Access Swagger UI
# Browser: http://localhost:8080/swagger

# Stop
docker-compose -f docker-compose.dev.yml down
```

## 🆘 Troubleshooting Quick Links

| Issue | Solution |
|-------|----------|
| Port 8080 in use | `Get-NetTCPConnection -LocalPort 8080` |
| Database won't start | `./docker-manage.ps1 dev-logs mssql` |
| API can't reach DB | Check network: `docker network inspect patient-portal-network` |
| Migrations failing | Manually run: `./docker-manage.ps1 migrate` |
| Complete reset | `./docker-manage.ps1 clean-all` then `dev-start` |
